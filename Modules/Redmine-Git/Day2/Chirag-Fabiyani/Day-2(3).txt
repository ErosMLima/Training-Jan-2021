Fetch:-
	Fetch helps in fetching the changes done on the remote server and compare it to the local server files
	before updating the local repo.
Pull:-
	Pull is used to update the files in the local repository and synchronize it with the remote repository.
	sometimes due to changes in both the repositories conflicts could occur.
Stage:-
	Stage is used to organize the files which are required to be commited to the remote server.
	All those files which are ready to commit are moved to stage and the remaining unstaged files are not commited
	unless the user wants.
Stash:
	Stash is used to keep the files which are modified into the stash area to remove conflicts and after apllying
	the stash both the local and remote repository files are merged.
